# 算法的概念

定义[课本]：算法是指解决问题的一种方法或一个过程，更严格的讲，算法是由若干条指令组成的有穷序列

# 算法的四条性质

  • 输入：有零个或多个由外部提供的量作为算法的输入
  • 输出：算法产生至少一个输出
  • 确定性：组成算法的每条指令是清晰无歧义的
  • 有限性：算法中每条指令的执行次数是有限的，执行每条指令所需要的时间也是有限的

# 实际计算机中运行时所耗费的资源

算法的复杂性是算法运行所需要的计算机资源的量，需要时间资源的量称为时间复杂性，需要空间资源的量称为空间复杂性

# 抽象计算机模型

为了衡量算法复杂性，并集中反应算法的效率，应该将算法从实际计算机中抽象出来，只依赖于要解问题的规模 N、算法的输入 I和算法 本身A的函数。 $$ C = F(N, I, A) $$

  • 通常来讲，A隐含在上下文中，通常被省略，得到时间复杂性和空间复杂性函数

$$ T = T(N, I)\quad S = S(N, I) $$

  • 时间复杂性可以进一步分解为抽象计算机上的元运算执行时间的和 $$ T(N, I) = \sum_{i = 1}^{k}t_ie_i(N, I) $$ 其中k为抽象计算机提供元运算个个数，ti，ei分别为执行元运算需要的时间以及在一次运行中执行的次数

  • 一般来讲，我们需要根据具体问题来对N，T的取值作出定义，只选取重要的指标参与计算。例如，在排序算法中只选取比较次数作为T的指标，等。

# 规模N的函数

显然，我们不能对每一种合法输入I都统计其ei的值，因此通常选取某类代表性的合法输入来作为代表进行计算。一般情况下，我们选取最坏情况下的输入 $$ T_{max} = \max_{I\in D_N}\sum_{i=1}^{k}t_ie_i(N, I) = T(N,I^*) $$

# 渐进函数

当不同算法的效率差异较大时，我们可以进一步对复杂性函数进行抽象，只考虑其阶的大小。常用的记号有 $$ O, \Theta, \Omega, o, \omega $$
